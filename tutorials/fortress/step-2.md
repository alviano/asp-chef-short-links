# How to start

Let's start by understanding the input format.

## Input format

The first line contains two integers, the number `R` of rows, and the number `C` of columns in the map.
The following `R` lines contain `C` integers each, denoting ordinary cells (`0`) or point-of-interests (`D`, with `D > 0`).

## Parsing

Copy the CSV content in the **Input panel**, and click the **ENCODE** button.
You should see a Base64-encoded content in the **Output panel**.

Now add a **Parse CSV** ingredient and specify **SPACE** as the separator.
The output is now in a format we can easily process with ASP rules:
```asp
__cell__(1,1,23).
__cell__(1,2,23).
__cell__(2,1,0).
__cell__(2,2,0).
__cell__(2,3,0).
...
```
